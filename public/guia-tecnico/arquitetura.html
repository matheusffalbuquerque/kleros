🏢 Arquitetura Multi-tenant

Isolamento de dados por congregação

ESTRATÉGIA: Database Shared + Tenant Identification

1. IDENTIFICAÇÃO DE TENANT:
   ├── Subdomínio → {congregacao}.local
   ├── Middleware AcessarCongregacaoPeloDominio
   ├── Binding no Service Container: app('congregacao')
   └── Contexto disponível em toda aplicação

2. ISOLAMENTO DE DADOS:
   ├── Foreign Keys: congregacao_id em tabelas principais
   ├── Global Scopes (quando necessário)
   ├── Policy-based authorization
   └── Context injection nos controllers

3. FLUXO DE REQUISIÇÃO:
   Request → Middleware → Identify Tenant → Bind Context → Continue
Componente Responsabilidade Implementação
Middleware Identificar tenant AcessarCongregacaoPeloDominio
Service Container Fornecer contexto app('congregacao')
Models Filtrar dados Foreign keys + Scopes
Controllers Aplicar contexto Injeção de dependência

Service Provider Pattern

class CelulasServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        $this->loadRoutesFrom(__DIR__ . '/../routes/web.php');
        $this->loadViewsFrom(__DIR__ . '/../resources/views', 'celulas');
        $this->loadMigrationsFrom(__DIR__ . '/../database/migrations');
    }
}

🗄️ Camada de Dados

Estratégia de persistência e acesso a dados

Padrão Repository (Implícito)

Eloquent ORM atua como Repository Pattern

// Model como Repository
class Membro extends Model
{
    // Query Scopes
    public function scopeAtivos($query) {
        return $query->where('ativo', true);
    }
    
    // Relationships
    public function congregacao() {
        return $this->belongsTo(Congregacao::class);
    }
    
    // Business Logic
    public function getNomeCompletoAttribute() {
        return $this->primeiro_nome . ' ' . $this->sobrenome;
    }
}

🔐 Arquitetura de Segurança

Camadas de proteção e autorização

CAMADAS DE SEGURANÇA:

1. NETWORK LEVEL:
   ├── Domain-based isolation
   ├── HTTPS enforcement
   └── CORS policies

2. APPLICATION LEVEL:
   ├── Authentication (Laravel Sanctum)
   ├── Authorization (Spatie Permission)
   ├── CSRF Protection
   └── Input Validation

3. DATABASE LEVEL:
   ├── Eloquent Mass Assignment Protection
   ├── SQL Injection Prevention
   ├── Foreign Key Constraints
   └── Data Encryption (quando sensível)

4. BUSINESS LOGIC LEVEL:
   ├── Role-based Access Control (RBAC)
   ├── Tenant Data Isolation
   ├── Activity Logging
   └── Policy-based Authorization
Nível Tecnologia/Padrão Implementação
Autenticação Laravel Auth + Session Multi-domain sessions
Autorização Spatie Permission Roles: kleros, admin, gestor
Validação Form Requests Request validation classes
Middleware Laravel Middleware Domain, Auth, Role checks
